RankED — Class Diagrams

Three focused diagrams: Strategy pattern · Observer pattern · Factory pattern · Data models

Diagram 1 — Strategy Pattern (ScoringStrategy.js)

Why Strategy? Brute-force would be one giant if-else inside submitAnswer() — every new exam type means touching GameEngine. Strategy extracts each algorithm into its own class behind a common interface (IScoringStrategy). GameEngine calls strategy.calculate() without knowing which strategy it has — that's OCP (open for extension, closed for modification) and polymorphism in action. ScoringStrategyResolver acts as the single factory entry point — callers never call new JEEMainStrategy() directly (DIP).
Diagram 2 — Observer Pattern (LeaderboardObserver.js)

Why Observer? Brute-force: submitAnswer() directly calls io.emit() — GameEngine is coupled to socket.io and any future analytics. Observer decouples them: BattleSubject just calls notify(), and any number of observers react independently. New observer (e.g. AnalyticsObserver) added without touching GameEngine — pure OCP. Each observer has SRP: LeaderboardObserver only manages leaderboard state and broadcast.
Diagram 3 — Factory Pattern (QuestionFactory.js)

Why Factory? Three question types (MCQ, Integer, MultipleCorrect) each validate answers differently. Brute-force: scattered if type === "MCQ" blocks everywhere in routes and GameEngine. Factory gives a single entry point QuestionFactory.create(type, data) — caller gets back any question object and just calls .validate(answer). This is abstraction (hide which class is instantiated), polymorphism (all share same interface), and DIP (callers depend on the factory abstraction, not concrete classes).
Diagram 4 — Data Models + Core Services (models + GameEngine + SocketManager)

Key relationships: GameEngine uses (dependency) all three patterns — it holds a Map of active match states, each containing a resolved IScoringStrategy and a BattleSubject. SocketManager owns (composition) a GameEngine — one GameEngine per server. Mongoose models are shown as classes with their fields and notable methods (comparePassword, getPublicProfile, getRandomQuestions).
